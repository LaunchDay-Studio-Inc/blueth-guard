# Security Policy — Blueth Guard

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.0.x   | ✅ Yes    |

## Reporting a Vulnerability

We take security seriously. If you discover a vulnerability in Blueth Guard, please report it responsibly.

### How to Report

**Email:** info@launchdaystudio.com

Please include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### What to Expect

- **Acknowledgment:** Within 48 hours of your report
- **Initial assessment:** Within 1 week
- **Fix timeline:** Critical vulnerabilities within 2 weeks, others within 1 month
- **Credit:** We will credit you in the release notes (unless you prefer anonymity)

### Scope

The following are in scope:
- The Blueth Guard Android application
- Build and release pipeline security
- Dependency vulnerabilities

The following are out of scope:
- Social engineering attacks
- Denial of service attacks
- Issues in third-party dependencies that are already publicly known

### Responsible Disclosure

Please do not:
- Publicly disclose the vulnerability before we've had a chance to fix it
- Access or modify other users' data
- Perform actions that could harm the availability of the service

## Security Best Practices

Blueth Guard follows these security practices:
- All dependencies are regularly audited
- The app requests only the minimum permissions necessary
- No data is transmitted to external servers
- Code is open source for community review
- Release APKs are signed with a secure key
